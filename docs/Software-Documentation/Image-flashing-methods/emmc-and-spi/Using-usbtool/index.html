Flashing using usb_tool

Overview​

Once power is available, the board will read firmware, bootloader and kernel from a boot device. The most common boot device used is eMMC and SPI flash. This section describes updating eMMC and SPI flash using usb_tool.

A typical image for SOM_SL1680_Mk1 is a directory SYNAIMG, that contains several subimg files and emmc_part_list, emmc_image_list and emmc_image_list_full. The emmc_part_list describes the GUID Partition Table (GPT) which will be used for the eMMC. The emmc_image_list* files specify which sub image files should be written to which partition on the eMMC.

Flashing from Linux host​

In order to flash via USB, you need to have usb_tool installed in your host system. This can be downloaded from here.

After downloading, copy the image folder (SYNAIMG) into the usb_tool directory.

  • Pre-built eMMC images can be obtained from Sentinel SDK Releases page.
  • Pre-built SPI images can be obtained from SPI U-boot Releases page.

Steps to update eMMC​

  1. Ensure the eMMC image is present in the usb-tool directory (named eMMCimg for pre-built or SYNAIMG for custom builds).
  2. Run the update_emmc script to begin the eMMC update.

Steps to update SPI flash​

  1. Copy sl1680 folder from SPI U-boot Releases into the usb_tool folder.
  2. Run the update_spi script to begin the spi update.

A binary named astra_update gets executed each time when update_emmc or update_spi is executed. The source code for this binary can be found at this here.

In order to start the update process, SOM_SL1680_Mk1 board needs to be set in the USB_BOOT mode. To do this, press and hold the USB_BOOT button and reset the board by pressing and releasing the reset button.

note

Ensure the USB_BOOT button is pressed long enough after the reset button is released. This is to ensure that the device detects the USB_BOOT mode. Also ensure that the board is in eMMC mode.

After booting into USB Boot mode, U-Boot will automatically flash the eMMC/SPI image from the host onto the SOM_SL1680_Mk1 board. The board will automatically reboot when the update is complete.

Flashing from a Windows PC​

SOM_SL1680_Mk1 board can also be updated through a Windows host PC via usb_tool. A USB driver is required to be installed in the host PC for communicating with the board. For that, install the usb_tool as specified previously.

Inside the Synaptics_WinUSB_Driver directory, right click on SYNA_WinUSB.inf and select install from the dropdown menu.

Click on install in the submenu for installing the driver.

note

Installing the Windows driver requires an account with administative privileges. Please contact your System Administrator if you do not have sufficient privileges.

After installing the driver, SOM_SL1680_Mk1 board will show up in Windows Device Manager as “Synaptics IoT: Tools package USB Driver for Synaptics Processors” when operating in USB Boot mode.

note

SOM_SL1680_Mk1 board won't be detected by the tool or be displayed in the device manager, if the device is not in USB_Boot mode.

Setup the hardware as discussed above.
