Building with temporary changes

This page describes the steps to take a build with temporary changes in the SDK for development purpose.

An overview of Yocto SDK​

Yocto Project is an open-source build system and toolset for creating custom Linux distributions for embedded devices. The yocto SDK comprises of layers. Each layer contains code for a specific purpose, it might be board specific or general.

Each layer contains multiple recipes. A recipe is a file that describes how to build a particular software component. In order to change a software source code, you need to edit the recipe file associated with it.

Adding changes in SDK for development purpose​

Yocto provides a command-line tool called devtool to add as well as modify existing recipes and source code. This way, the developer can make changes to the code temporarily without editing the actual recipe. The usage of devtool can be demonstrated with an example :

Suppose you need to change the model name of the root node in the device tree source from Sentinel_sl1680 RDK to Sentinel_board RDK. You will have to modify the recipe associated with the linux source. linux_dct is the recipe that handles the linux source code.

Follow the below procedure to change the model name :

  1. Setup the environment before running the devtool utility:
docker run --rm -it -v $(pwd):$(pwd) ghcr.io/synaptics-astra/crops:1.0.0 --workdir=$(pwd)
MACHINE=sentinel_som_sl1680_mk1 source meta-dctk-sentinel/setup/setup-environment
  1. To prepare the recipe for modifying the source code, run
devtool modify linux-dct

After executing this command, a workspace directory will be created inside the <build-dir> folder. The source code for linux kernel will be obtained inside the folder <build-dir>/workspace/sources/linux-dct/

  1. Rename the model in the dts file as below

  2. You can build the image from here in two ways, either through bitbake utility or continue with the devtool

    1. To build using bitbake, execute the following commands :
    bitbake -f linux-dct -c compile
bitbake -f sentinel-media
    1. To build using devtool, execute the following commands :
    devtool build linux-dct
devtool build-image sentinel-media

Run the appropriate image as per your requirement. Here, we used the sentinel-media as an example.

If you are satisfied with testing using devtool, you can go back to compiling from the actual recipe by executing the command

devtool reset linux-dct

This will set bitbake to complile the source code of meta-dctk-sentinel instead of workspace. This method of customizing and building is for development purposes. Next topic guides you to integrate those changes permanently into meta-dctk-sentinel .

note

When devtool is active, bitbake overrides the actual recipe and looks into the recipe source in the workspace folder. In order to use the actual recipe present in meta-dctk-sentinel, you must execute the reset command.
