Boot Process

The Sentinel SOM SL1680 MK1 supports two distinct boot modes, allowing flexibility based on storage preference and application requirements:

  • SD Card mode

In this mode, the system boots directly from a microSD card inserted into the designated slot. This method is ideal for development, testing, or scenarios where storage needs to be easily removable or replaceable.

  • eMMC mode

This mode enables the system to boot from its onboard embedded MultiMediaCard (eMMC) storage. It is typically used for production environments where more permanent and reliable storage is required.

The way of booting can be controlled by SW1 switch as shown in the below figure.

SD card mode​

When SW1 is in ON position, the board will boot in SD card mode. In this mode, the device will load the U-boot stored in the internal SPI flash and will boot up the linux kernel and rootfs present in the SD card. This mode requires an SD card compatible image to be flashed in the SD card which is discussed in this section.

If no SD card is found, the device will stay in the command line of U-boot.

eMMC mode​

When SW1 is in OFF position, the board will boot in eMMC mode. The device will load the U-boot stored in the eMMC and will further boot up the Linux kernel and rootfs present in eMMC.
