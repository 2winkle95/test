HDMI RX

Let's verify HDMI RX.

Getting Started​

This guide explains how to test the HDMI Receiver (RX) interface on the Sentinel SOM_SL1680_Mk1. Follow the steps below to verify HDMI signal integrity, resolution handling, and audio/video passthrough.

1. Prerequisites​

Before starting HDMI RX/TX testing, ensure the following:

  • HDMI source (e.g., laptop, media player).
  • HDMI sink/display (e.g., monitor, TV).
  • Proper HDMI cables (High-Speed recommended).
  • Access to the target board console.

2. Interface Testing.​

2.1 Connect the HDMI Source​

  • Connect the HDMI source device to the HDMI IN (J9) port of the board.

2.2 Capture Video Stream​

Gstreamer can use the V4L2 interface to process and display the video and audio streams received from the external HDMI device.

The V4L2 device file number may vary depending on your configuration. You can use the v4l2-ctl command to find which device files are associated the HDMI-RX device.

v4l2-ctl --list-device

Supported Formats​

Typical PC Video Formats:​

FormatBits per ChannelResolution
RGB8, 10, 12 bits4K60
YUV4228, 10, 12 bits4K60
YUV4448, 10, 12 bits4K60

V4L2 Video Formats:​

FormatResolution
NV124K60
UYVY4K60

You can use the following commands to route the HDMI RX video to the HDMI sink.

This example displays a 4K30 stream from an external HDMI device using waylandsink:

gst-launch-1.0 v4l2src device=/dev/video6 ! video/x-raw,width=3840,height=2160,fps=30,format=NV12 ! waylandsink fullscreen=true
note

For more Gstreamer commands see Gstreamer command line tools

2.3 Check Resolution and Format​

  • Read HDMI input format:
cat /sys/class/drm/card0-HDMI-A-1/modes
  • Ensure the captured resolution matches the input signal.

2.4 Audio Validation​

Audio Formats​

ChannelsSample RateBit Depth
248kHz32 bit

Use ALSA utilities for capturing the audio from HDMI-RX:

Use the arecord -l command to determine which ALSA capture device is associated with HDMI-RX.

Use the aplay -l command to determine which ALSA playback device to use to play the captured audio. The following examples will use the speakers associated with the HDMI sink.

After identifying the correct device please run the following command for routing the audio:

gst-launch-1.0 alsasrc device=hw:0,8 ! queue ! audio/x-raw,format=S32LE,rate=48000,channnels=2 ! alsasink device=hw:0,7 sync=false
