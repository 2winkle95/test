MIPI CSI

Let's verify MIPI CSI.

Getting Started​

Sentinel SOM_SL1680_Mk1 support 2 MIPI CSI ports, CSI0 and CSI1.

  • CSI0 - 4 lane - 22 pin
  • CSI1 - 2 lane - 15 pin

Image Sensor Cameras​

Sentinel SOM_SL1680_Mk1 includes an integrated ISP and supports connecting image sensor camera modules using the MIPI-CSI connectors. Gstreamer can use these cameras using the V4L2 interface. The ISP supports 3 output paths, the main path supports outputing 4K resolution (if the sensor supports 4K), and the Secondary Paths support 2K resolution. Each path has it’s own video device file in /dev.

The device file number may vary depending on your configuration. You can use the v4l2-ctl command to find which device files are associated with each of the ISP paths.

v4l2-ctl --list-device

An additional video device is created to support Bayer RGB capture. This is the 4th video instance which advertises itself as only supporting Bayer RGB. The other 3 video ports support NV12 / RGB only.

To display video from the ISP’s Main Path:

gst-launch-1.0 v4l2src device=/dev/video3 ! 'video/x-raw, format=(string)NV12, width=(int)640, height=(int)480, framerate=(fraction)60/1' ! waylandsink

To display video from the ISP’s Secondary Paths change the device in the command accordingly.

Multi Stream Support​

Sentinel SOM_SL1680_Mk1 supports up to three video streams to be enabled simultaneously (excluding the Bayer RGB video port - vvcam-video.0.3). Up to three different GStreamer pipelines can be run at the same time to validate this feature any combination of up to 3 paths — Main, Secondary Path 1, and Secondary Path 2 can be run simultaneously.

note

Although multiple streams may appear as independent devices, all streams originate from a single instance. Once all streams are playing, any change made to an existing stream is treated as a new change. Hence, ensure that all sessions are closed one by one before starting the next set of multi or single streaming.

Dual Sensor Support​

Sentinel SOM_SL1680_Mk1 supports dual sensor configuration with the OV5647 sensor module. Dual sensor support required enabling the Sentinel-bothcsi-without-expander.dtbo overlay. (See Updating the ISP Sensor Configuration). Once the two sensors are connected and the overlay is enabled, v4l2-ctl will display a total of eight vvcam-video devices- three NV12 / RGB devices plus one Bayer RGB device per sensor.

Similar to multi stream support, multiple Gstreamer pipelines can be used with both sensors simultaniously.

For example, these commands will display video from the main path of the sensor connected to CSI0 and video from the main path of the sensor connected to CSI1.

gst-launch-1.0 v4l2src device=/dev/video3 ! 'video/x-raw, format=(string)NV12, width=(int)640, height=(int)480, framerate=(fraction)30/1' ! waylandsink

gst-launch-1.0 v4l2src device=/dev/video7 ! 'video/x-raw, format=(string)NV12, width=(int)640, height=(int)480, framerate=(fraction)30/1' ! waylandsink
note

IMX258 and IMX415 sensors are not supported with the dual sensor configuration on Sentinel SOM_SL1680_Mk1 boards. Both sensors require a GPIO expander whereas CSI1 is incompatible with the GPIO expander.

Updating the ISP Sensor Configuration​

By default, Sentinel SOM_SL1680_Mk1 is configured to use the OV5647 module with port CSI0. But, the software can be configured to use any of the supported sensor modules. Some configurations require setting a device tree overlay to function correctly. We verified both the CSI0 and CSI1 using OV5647 interface.

Steps to verify the CSI interface using OV5647​

Overlays to by used​

SensorModuleInterfaceDevice Tree Overlay
OV5647Arducam 5MP OV5647 Camera ModuleMIPI-CSI 0
MIPI-CSI 1
Dual CSI0/1		N/A
sentinel-sl1680-csi1-without-expander.dtbo
sentinel-sl1680-bothcsi-without-expander.dtbo

1. Updating Device Tree Overlay​

  • Boot into U-Boot by continuosly pressing any key while booting and set variable dtbo with the required device tree overlay.

For example to use the OV5647 camera module in CSI1 port:

setenv dtbo sentinel-sl1680-csi1-without-expander.dtbo
  • Verify the variable is correctly set in the U-Boot.
printenv
  • Save the variable to the eMMC for persist in reboot.
saveenv
  • Finally boot the device with updated overlay.
boot

2. Changing the Camera Sensor Module​

The file /proc/vsi/isp_subdev0 is used to display the sensor configuration properties and to update the sensor configuration.

1. Read the current sensor configuration.​

cat /proc/vsi/isp_subdev0

2. Configuring the new module​

Configuring the new module by editing /usr/sbin/isp_media_server.sh script. Change the line which writes the sensor config to /proc/vsi/isp_subdev0. Update the sensor name, xml, and json files for the new sensor.

To list the available xml and json files for sensor please use:

ls /usr/share

For example to use the OV5647 camera module in CSI1 port:

Then restart the isp_media_server service to apply the update.

systemctl restart isp_media_server

3. Getting the camera stream​

We can use the Gstreamer utility for getting the video stream from the camera module.

gst-launch-1.0 v4l2src device=/dev/video3 ! 'video/x-raw, format=(string)NV12, width=(int)640, height=(int)480, framerate=(fraction)30/1' ! waylandsink
info

Please follow the steps to described above to identify the /dev/video device corresponding to the camera module.
