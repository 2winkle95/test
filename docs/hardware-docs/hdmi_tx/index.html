HDMI TX

Let's verify HDMI TX.

Getting Started​

This guide explains how to test the HDMI Transmitter (TX) interface on Sentinel SOM_SL1680_Mk1. Follow the steps below to verify HDMI signal integrity, resolution handling, and audio/video passthrough.

1. Prerequisites​

Before starting HDMI RX/TX testing, ensure the following:

  • HDMI source (e.g., laptop, media player).
  • HDMI sink/display (e.g., monitor, TV).
  • Proper HDMI cables (High-Speed recommended).
  • Access to the target board console.

2. Video sinks available​

Gstreamer on Sentinel SOM_SL1680_Mk1 supports three video sinks.

  1. The main video sink is the waylandsink which uses the wayland protocol and compositor to display the video output.
  2. Sentinel SOM_SL1680_Mk1 also supports the DRM KMS sink which displays video frames directly to a Linux DRM device using the kmssink.
  3. The xvimagesink is supported when Sentinel SOM_SL1680_Mk1 is running an image with the X11 based display server.

2.1 Wayland Sink​

Many of the following examples use the Wayland video sink to create a window and render the decoded frames (see GStreamer documentation for more details)

When using the Wayland sink, please make sure to set the following variables in your environment. These variables may need to be set when running commands from the serial console or a remote shell:

export XDG_RUNTIME_DIR=/var/run/user/0
export WAYLAND_DISPLAY=wayland-1

The XDG_RUNTIME_DIR variable specifies the directory which contains the Wayland socket belonging to the user. The WAYLAND_DISPLAY variable specifies which Wayland compositor to connect to.

2.2 KMS Sink.​

The KMS sink supports displaying video without the need to run a windowing system like Wayland, since it can interface directly with Linux DRM devices. (see GStreamer documentation for more details)

Before using kmssink be sure to disable the Wayland service if it is running:

systemctl stop weston.service

You will also need to identify the plane id for the main plane and provide this id to the kmssink element. You can identify the plane id using the modetest command:

modetest -M synaptics

Identify the plane id of the plane which supports formats: NV12 NV21 UYVY VYUY YUYV YVYU

2.3 XvImageSink.​

The XvImage sink supports displaying video using the X11 backend. This can be used by, replacing waylandsink with xvimagesink in the Gstreamer commands. (see GStreamer documentation for more details).

3.HDMI configuration​

HDMI configuration options in dts are as below:

hdtx-core-config: Used to configure HDMI output, it includes bitfields to control below options.

Default options:hdtx-core-config = /bits/ 8 <1 0 1>

  • HPD handling: To enable/disable handling of sink hotplug. If HPD handling is disabled, output format will be configured on bootup depending on sink capability and retained till next reboot.

  • HDCP control: To enable/disabled HDCP. Currently not handled.

  • FixedModeset: To let SYNA DRM/KMS driver handle the mode setting internally without exposing user interface. If set, it will configure output format without requiring the userspace application to configure output format, any attempt to override the configured format will be ignored.

hdtx-supported-formats: Used to list the formats exposed to user via kms mode query.

Default selection: hdtx-supported-formats = /bits/ 8 <12 9 10 13 22 21 19 26 25 24 61 62 64>

Refer below file for resolution indices in

'build/workspace/sources/linux-dct/drivers/soc/berlin/modules/avio/vpp/ca/include/vpp_defines.h'

If needed, this can be overridden using parameter hdmi_preferred_mode, by adding/modifying the same in /etc/modprobe.d/syna_drm.conf.

Example: To override preferred modes as 1080p: hdmi_preferred_mode=1920x1080

Below is out of box default configuration,

  • syna drm/kms driver will internally configure HDMI output format.
    • On connecting 4K supported sink, 4K30 RGB 8bit format will be selected.
    • On 1080p TV, output will be configured as 1080p60 RGB 8bit.
  • Preferred mode of EDID is override to 1080p.
note

For more informations refer general display configuration modes.

For HDMI display ID, depending on the capability of the SINK at bootup only 4K or 1080P are supported at present.

4. Interface Testing​

4.1 Connect the HDMI Sink​

  • Connect the HDMI sink device to the HDMI OUT/TX (J5) port of the board.

4.2 Verify Signal Detection​

  • On the console, check logs using:
dmesg | grep -i hdmi

or

cat /sys/class/drm/card0-HDMI-A-1/status

Expected output:

connected

Now you can route video to any of the available sinks available using Gstreamer utility (Refer Playing a video file for more commands).

Also the SYNA DRM/KMS driver testing can be done by:

1. Disable Weston

systemctl stop weston.service

2. Identify the plane id for the main plane

modetest -M synaptics

3. Use Modetest tool to route a sample pattern to hdmi-tx

modetest is a tool provided by libdrm library and is available as part of the SDK release/image (/usr/local/bin/modetest). Following are some of the tasks performed with modetest

  • List all display capabilities: CRTCs, encoders & connectors (DP, HDMI, DSI …), planes, modes…
  • Perform basic tests: display a test pattern, display 2 layers, perform a vsync test.
  • Specify the video mode: resolution and refreshrate.

Below is the syntax.:

modetest -M synaptics -s <connector_id> [,
<connector_id>][@<crtc_id>]:[#<mode index>]<mode>[-<vrefresh>][@<format>]

Example: modetest -M synaptics -s 43:1920x1080
