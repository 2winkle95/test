SPI

Let's verify SPI.

Getting Started​

Sentinel SOM_SL1680_Mk1 has two SPI interfaces. One is connected to the internel spi flash (16 Mb) and the second one is connected to the 40-pin GPIO header. This can be verified by connecting an externel nor flash to the SPI lines and accessing the same from user space with spidev_test tool.

SPI Lines on 40 pin header​

Pin NoName
23SM_SPI2_SS1_LV
24SM_SPI2_SDI_LV
25SM_SPI2_SS0_LV
27SM_SPI2_CLK_LV
28SM_SPI2_SS3_LV
30SM_SPI2_SDO_LV

Adding spidev_test tool to the build image​

To include the spidev_test user-space tool in your Yocto image.

Steps:​

  1. Create a new recipe
mkdir -p meta-dctk-sentinel/recipes-tools/spidev-test/files
  1. Copy the source file from the kernel to your recipe
cp build/workspace/sources/linux-dct/tools/spi/spidev_test.c meta-dctk-sentinel/recipes-tools/spidev-test/files/
  1. Create the recipe file spidev-test_1.0.bb

Path: meta-dctk-sentinel/recipes-tools/spidev-test/spidev-test_1.0.bb

DESCRIPTION = "SPI Device Test Tool"
LICENSE = "GPL-2.0"
LIC_FILES_CHKSUM = "file://${COMMON_LICENSE_DIR}/GPL-2.0-only;md5=801f80980d171dd6425610833a22dbe6"

SRC_URI = "file://spidev_test.c"

S = "${WORKDIR}"

do_compile() {
    ${CC} ${CFLAGS} ${LDFLAGS} -o spidev_test spidev_test.c
}

do_install() {
    install -d ${D}${bindir}
    install -m 0755 spidev_test ${D}${bindir}
}
  1. Include the tool in your image

Add the image in meta-dctk-sentinel/recipes-bsp/images/sentinel-media-oobe.bb.

IMAGE_INSTALL:append = " spidev-test"
  1. Build the image

From docker run the commands

bitbake spdev-test
bitbake <your-image-name>

This will install spidev_test to /usr/bin/ in the final image.

note

You can also build the spidev_test tool directly from the kernel source instead of creating a separate recipe. Please modify the kernel recipe meta-dctk-sentinel/recipes-kernel/linux/linux-dct_5.15.140.bb
